협업을 위한 도구와 클라우드 서비스

###  Git & GitHub (버전 관리 및 협업 기초)
깃허브 데스크톱을 활용하여 프로젝트의 형상을 관리하고, 팀원들과 원활하게 코드를 공유

#### 1. 커밋 메시지 규칙
어떤 변경 사항인지 알 수 있도록 규칙에 따라 메시지를 작성
- **Feat:** 새로운 기능 추가
- **Fix:** 버그 수정
- **Style:** 오타 등 코드에 영향을 주지 않는 변경
- **Refactor:** 코드 리팩토링
- **Docs:** 문서 수정

#### 2. 형상 관리 과정
- **Commit:** 변경된 파일을 저장
- **Push:** 커밋 내역을 깃허브 저장소로 업로드
- **Pull:** 팀원이 올린 코드를 내 로컬로 내려받기

#### 3. 충돌(Conclict) 해결
여러 명이 같은 파일을 수정할 때 발생하는 충돌 상황이 있음
- **해결:** Pull 과정에서 코드가 겹칠 때, 깃허브 데스크톱이 제공하는 툴을 활용하여 반영할 코드를 직접 선택 및 제외
- **Stash:** 코드를 다 커밋하지 않은 상태에서 Pull을 해야 할 때, 현재 작업 중인 코드를 임시 저장하고 나중에 다시 불러옴

### AWS 계정 생성 및 보안 설정 (MFA)
개인적으로 직접 AWS 계정을 생성하고 보안을 설정해봤음

- **AWS 계정 생성:** AWS 프리티어 계정을 생성하여 클라우드 환경에 접근
- **2단계 인증(MFA) 설정:** 루트 계정의 탈취를 막기 위해 Google Authenticator을 활용한 MFA를 설정
- **의의:** 보안 과금 사고나 해킹을 방지



### Docker & Kubernetes 개념 학습

- **Docker:** 어디서나 동일하게 작동하는 환경(컨테이너)을 만드는 기술
- **Kubernetes:** 분산된 컨테이너들을 관리하는 도구
