효율적인 협업을 위한 버전 관리 도구와 클라우드 서비스의 보안 기초 사항을 정리합니다.

### 1. Git & GitHub Desktop
명령어(CLI)가 익숙하지 않은 환경에서도 **GitHub Desktop** GUI 도구를 활용하여 프로젝트의 형상을 관리하는 법을 익혔습니다.

- **버전 관리의 흐름:** 코드 수정 후 변경 사항을 확인하고, 의미 있는 단위로 **Commit**하여 기록을 남깁니다.
- **원격 저장소 동기화:** 로컬의 작업 내용을 **Push**하여 GitHub에 백업하고, 팀원과 코드를 공유할 수 있는 기초 체력을 길렀습니다.

### 2. AWS 계정 생성 및 보안 설정 (MFA)
클라우드 서비스 이용의 핵심인 계정 보안을 직접 실습해 보았습니다.

- **AWS 계정 생성:** AWS 프리티어 계정을 생성하여 클라우드 컴퓨팅 환경에 접근해 보았습니다.
- **2단계 인증(MFA) 설정:** 루트 계정의 탈취를 막기 위해 Google Authenticator 등을 활용한 **MFA(Multi-Factor Authentication)**를 설정했습니다.
- **의의:** 실무에서 발생할 수 있는 보안 과금 사고나 해킹을 방지하기 위한 보안 절차의 중요성을 체감했습니다.



### 3. Docker & Kubernetes (개념 학습)
향후 학습할 현대적 배포 기술의 개념을 정리했습니다.

- **Docker:** 어디서든 동일하게 작동하는 환경(컨테이너)을 만드는 기술.
- **Kubernetes:** 분산된 컨테이너들을 체계적으로 관리하는 오케스트레이션 도구.
- **학습 목표:** 현재는 개념 위주로 파악하고 있으나, 추후 프로젝트 규모 확장 시 실제 배포 파이프라인에 적용해 볼 계획입니다.
