리액트는 사용자 화면(프론트)을 위한 JavaScript 라이브러리로, 부드러운 화면 전환과 재사용성을 위해 컴포넌트 기반의 구조를 가진다

### 1. Component
컴포넌트는 화면 구성하는 하나의 단위라고 할 수 있는데, 독립적이고 재사용 가능하다.
- **특징:** 레고처럼 조립하여 화면을 구성함
- **참고:** 현재는 **함수형 컴포넌트(Functional Component)**와 **Hooks**를 사용하는 것이 표준(과거에는 클래스형 컴포넌트 사용했었다고 함)

### 2. Props
부모 컴포넌트가 자식에게 전달하는 데이터
- **특징:** 읽기 전용 데이터라서 자식이 직접 수정할 수 없음
- **사용:** 컴포넌트 간에 정보를 전달할 때

### 3. State
컴포넌트 내부에서의 동적인 데이터
- **특징:** 상태값이 변경되면 리액트가 감지하고 그 컴포넌트를 **리렌더링**함
- **관리:** `useState` 훅을 사용하여 상태를 업데이트
