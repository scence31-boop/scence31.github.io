웹 개발 중 브라우저 개발자 도구의 네트워크 탭에서 마주하는 HTTP 상태 코드로, 프로젝트를 진행하며 경험한 에러 유형

### HTTP 상태 코드
- **200:** 성공
- **4xx:** 클라이언트 오류, 프론트엔드 코드나 파라미터 확인 필요
- **500:** 서버 오류, 백엔드 로직 확인 필요 (주로 매핑 문제였음)



### 클라이언트 오류 (4xx)

| 코드 | 명칭 | 상황 |
| :--- | :--- | :--- |
| **400** | Bad Request | Ajax, Axios 데이터 전송 시 백엔드/프론트엔드 객체의 키값 불일치 |
| **401** | Unauthorized | **인증 없음.** 비로그인 상태이거나 세션/토큰이 만료된 경우 |
| **403** | Forbidden | **권한 없음(인가 실패).** 로그인은 되어있으나 해당 페이지에 접근 권한이 없음 |
| **404** | Not Found | URL 경로 오타 |
| **405** | Method Not Allowed | POST/GET 불일치 |

### 서버 오류 (500)

- **발생 원인:** 백엔드 로직 수행 중 예외(Exception) 발생
    - `NullPointerException`: 객체가 없는데 메소드 호출
    - `SQLException`: SQL 오류, DB 제약조건 위반

- **해결 방법:** 브라우저 콘솔이 아닌 STS의 Console 탭을 확인하여 에러가 발생한 라인을 찾아 수정



---

### 느낀점
400번대 에러는 네트워크 탭의 Payload(보낸 데이터)와 Preview(응답 메시지)를 보고 해결, 500 에러는 서버 로그를 분석해야 한다는 점을 배웠습니다.
